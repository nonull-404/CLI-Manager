# Version Update Checklist

> Purpose: Before changing the app version, read this checklist to update every project-level version source consistently.

## When to Use

- Preparing a new CLI-Manager release.
- User asks to bump the app version, for example `0.1.5` -> `0.1.6`.
- Tauri bundle metadata, npm metadata, or Rust package metadata must stay aligned.

## Version Sources to Update

| File | Field | Notes |
|------|-------|-------|
| `package.json` | top-level `version` | Frontend/npm project version. |
| `package-lock.json` | top-level `version` | Lockfile root metadata. Prefer regenerating with npm when possible. |
| `package-lock.json` | `packages[""].version` | Root package entry inside lockfile. Must match `package.json`. |
| `src-tauri/Cargo.toml` | `[package].version` | Rust crate/app package version. Do not touch dependency `version` fields. |
| `src-tauri/Cargo.lock` | `[[package]] name = "cli-manager"` version | Cargo lock root package metadata. Usually regenerated by `cargo check` after `Cargo.toml` changes. |
| `src-tauri/tauri.conf.json` | top-level `version` | Tauri bundle/app version shown to packaging/runtime metadata. |

## Safe Update Procedure

1. Search the current version string first:

```bash
grep -R "0.1.5" package.json package-lock.json src-tauri/Cargo.toml src-tauri/Cargo.lock src-tauri/tauri.conf.json
```

2. Update only the app/package version fields listed above.
3. Do not replace every `version` occurrence blindly; dependency declarations also contain `version` keys.
4. If `package-lock.json` is out of sync, prefer:

```bash
npm install --package-lock-only
```

5. Run `cargo check` or another Cargo metadata refresh after changing `src-tauri/Cargo.toml` so `src-tauri/Cargo.lock` syncs the root package version.
6. Verify the new version appears in all expected app-version places:

```bash
grep -R "0.1.6" package.json package-lock.json src-tauri/Cargo.toml src-tauri/Cargo.lock src-tauri/tauri.conf.json
```

## Current Gotcha

At the time this guide was added, the repo had mismatched version metadata:

- `package.json`: `0.1.5`
- `src-tauri/Cargo.toml`: `0.1.5`
- `src-tauri/tauri.conf.json`: `0.1.5`
- `package-lock.json` root entries: `0.1.4`

So future version bumps must explicitly fix both root entries in `package-lock.json`, or regenerate the lockfile.

## Acceptance Check

A version update is complete only when these values are identical:

- `package.json` top-level `version`
- `package-lock.json` top-level `version`
- `package-lock.json` `packages[""].version`
- `src-tauri/Cargo.toml` `[package].version`
- `src-tauri/Cargo.lock` `[[package]] name = "cli-manager"` version
- `src-tauri/tauri.conf.json` top-level `version`
